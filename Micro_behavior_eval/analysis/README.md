# Micro Behavior Eval Analysis

## Purpose
`Micro_behavior_eval/analysis` is a standalone analysis package for raw evaluation outputs generated by `Micro_behavior_eval`.

- Read-only input: `<eval_root>/<run_id>/micro_behavior_eval.csv` (default `eval_root=outputs/default/runs/source_default/micro_behavior_eval`)
- Write-only analysis output: `reports/default/micro_behavior/<analysis_run_id>/...`
- It does **not** write anything under the raw eval root (default `outputs/default/runs/source_default/micro_behavior_eval`).

## Run
From repo root:

```bash
python Micro_behavior_eval/analysis/run_analysis.py --run_id <eval_run_id>
```

Use a custom eval root (for benchmark outputs):

```bash
python Micro_behavior_eval/analysis/run_analysis.py \
  --eval_root outputs/benchmark/runs/benchmark_ood/all_or_nothing/false_to_true/micro_behavior_eval \
  --run_id oracle_archetype/<eval_run_id>
```

Or pass an explicit CSV:

```bash
python Micro_behavior_eval/analysis/run_analysis.py \
  --eval_csv outputs/default/runs/source_default/micro_behavior_eval/<eval_run_id>/micro_behavior_eval.csv \
  --analysis_root reports/default/micro_behavior \
  --analysis_run_id my_analysis_run
```

Compare multiple eval runs:

```bash
python Micro_behavior_eval/analysis/run_analysis.py \
  --eval_root outputs/default/runs/source_default/micro_behavior_eval \
  --compare_run_ids no_archetype/2602262019,oracle_archetype/2602270203,retrieved_archetype/2602270212 \
  --compare_labels "no archetype,oracle archetype,retrieved archetype" \
  --analysis_run_id compare_last_three
```

Four-way comparison (adds random archetype baseline):

```bash
python Micro_behavior_eval/analysis/run_analysis.py \
  --eval_root outputs/default/runs/source_default/micro_behavior_eval \
  --compare_run_ids no_archetype/2602262019,oracle_archetype/2602270203,retrieved_archetype/2602270212,random_archetype/2602270902 \
  --compare_labels "no archetype,oracle archetype,retrieved archetype,random archetype" \
  --analysis_run_id compare_last_four
```

Default-data (non-benchmark) example:

```bash
python Micro_behavior_eval/analysis/run_analysis.py \
  --eval_root outputs/default/runs/source_default/micro_behavior_eval \
  --compare_run_ids no_archetype/2602262019,oracle_archetype/2602270203,retrieved_archetype/2602270212,random_archetype/2602270902 \
  --compare_labels "no archetype,oracle archetype,retrieved archetype,random archetype" \
  --analysis_root reports/default/micro_behavior \
  --analysis_run_id source_default_last_four
```

## CLI Arguments
- `--eval_csv`: explicit input CSV path (required unless `--run_id` provided)
- `--run_id`: resolves `<eval_root>/<run_id>/micro_behavior_eval.csv`
- `--run_id` can include path segments (for example `oracle_archetype/2602270203`)
- `--eval_root`: run-root for `--run_id` resolution (default `outputs/default/runs/source_default/micro_behavior_eval`)
- `--compare_run_ids`: comma-separated run IDs for multi-run comparison mode
- `--compare_labels`: optional comma-separated labels aligned to `--compare_run_ids`
- `--analysis_root`: output root (default `reports/default/micro_behavior`)
- `--analysis_run_id`: output folder name (default timestamp)
- `--min_round`: optional lower round filter
- `--max_round`: optional upper round filter
- `--skip_no_actual`: `true|false`, default `true`
- `--dpi`: plot DPI, default `160`
- `--debug_print`: `true|false`, default `false`

## Output Structure
Under `reports/default/micro_behavior/<analysis_run_id>/`:

- Single-run mode:
  - `metrics_overall.csv`
  - `metrics_by_round.csv`
  - `metrics_by_game.csv`
  - `metrics_contribution_by_all_or_nothing.csv`
  - `row_level_scored.csv`
  - plot PNGs
  - `analysis_manifest.json`
- Comparison mode:
  - `comparison_overall.csv`
  - `comparison_by_round.csv`
  - `comparison_by_game.csv`
  - `comparison_contribution_by_all_or_nothing.csv`
  - `comparison_overall_errorbars.csv` (mean/SE/95% CI by run and metric)
  - `comparison_by_round_errorbars.csv` (mean/SE/95% CI by run-round and metric)
  - `comparison_significance.csv` (pairwise run differences, z-test p-values + BH-adjusted p-values)
  - `comparison_summary.md` (human-readable interpretation with plot guide + significance summary)
  - comparison plot PNGs
  - `analysis_manifest.json`

Error bars in comparison plots:
- Bars and lines now use mean ± 95% CI (`1.96 * SE`) computed from row-level scored data.
- Plot titles include direction cues (`↑ better` or `↓ better`).
- Run colors are assigned once per comparison run and reused consistently across all comparison figures (scales to >3 runs).

Significance in comparison mode:
- `comparison_significance.csv` runs pairwise two-sided z-tests on run-level mean differences.
- Includes raw `p_value` and Benjamini-Hochberg adjusted `p_value_bh`.
- Prefer `significant_bh_0_05` for multi-comparison interpretation.

## Metric Definitions
Contribution metrics:
- `predicted_contribution_parsed`: share with parseable contribution prediction
- `contrib_mae`: mean absolute error
- `contrib_rmse`: root mean squared error
- `contrib_bias`: mean(predicted - actual)
- `contrib_corr`: correlation (NaN-safe)

Contribution-regime metrics (`CONFIG_allOrNothing` stratification):
- `contrib_mae_norm20`: mean absolute error normalized by 20
- For `CONFIG_allOrNothing=true` rows only:
  - `contrib_binary_accuracy`
  - `contrib_binary_precision`
  - `contrib_binary_recall`
  - `contrib_binary_f1`
- Binary split rule: contribution `>= 10` is treated as `all`, `< 10` as `zero`.
- Regime stratification is populated from each run's `config.json -> inputs.analysis_csv` mapping (`gameId -> CONFIG_allOrNothing`).

Action metrics use typed targets:
- Punish target: `("P", playerId)`
- Reward target: `("R", playerId)`

Key distinction:
- Target metrics (`target_*`) evaluate who was targeted with type awareness, ignoring unit magnitude.
- Unit metrics (`unit_*_on_overlap`) evaluate amount quality only on overlapping typed targets.

Action metrics:
- `action_exact_match`: full typed target + unit exact match
- `target_set_exact`: exact typed target set match
- `target_precision`, `target_recall`, `target_f1`, `target_jaccard`
- `target_hit_any`: credits any correct typed target hit
- `unit_mae_on_overlap`, `unit_exact_on_overlap_rate`
- prevalence rates for any action, punish, and reward

## Guardrails
- Missing required columns -> fail-fast error
- Malformed dict strings -> coerced to `{}`, tracked in manifest
- Empty filtered data -> CSVs + manifest still written, plots skipped
- Correlation undefined -> NaN
- Safety check blocks output paths inside eval roots (including `outputs/default/runs/source_default/micro_behavior_eval`)

## Troubleshooting
- If you pass both no `--eval_csv` and no `--run_id`, the CLI raises an input error.
- If output path accidentally points under raw eval output, the CLI raises a safety error.
- If plots fail in headless environments, ensure matplotlib is available (`Agg` backend is used by default).
